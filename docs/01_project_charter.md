# 企画書

## プロジェクト名称

**Modrinth Mod Manager（mmm）**

## なぜ（目的）

### 現状の課題

Minecraft のバージョンアップに伴い、利用している MOD・shader・リソースパックの更新作業が必要になる。現在は以下の手順を手動で行っている：

1. Modrinth のサイトにアクセス
2. 約30個のプロジェクトを1つずつ検索
3. 各プロジェクトで Minecraft バージョンと Loader（Fabric）を指定
4. 対応バージョンが提供されているか確認
5. 提供されていればダウンロード

この作業は年に数回発生し、30個のプロジェクトに対して繰り返すため煩雑である。

### Modrinth 公式アプリを使わない理由

Modrinth 公式アプリ（Theseus）は多機能すぎる。Minecraft のインスタンス管理やランチャー機能など、不要な機能が多く含まれている。必要なのは「MOD 等の更新確認とダウンロード」だけである。

### 目的

**TOML ファイルで管理対象を定義し、コマンド一発で更新確認・ダウンロードを完了させる**

## 何を（作るもの）

### 作るもの

Modrinth API を利用した CLI ツール。以下の機能を提供する：

- TOML ファイルによる管理対象の定義
- 指定した Minecraft バージョン・Loader に対応するファイルの自動取得
- 更新があるかどうかの確認
- 差分ダウンロード（変更があったもののみ）

### 管理対象

| 種類 | 説明 |
|------|------|
| MOD | Fabric Loader 用の MOD |
| Shader | シェーダーパック |
| Resource Pack | リソースパック |

### 利用者

- 開発者本人（kk6）のみ
- 個人利用を前提とする

### 利用者が得られる便益

- 30個のプロジェクトの更新確認が数秒で完了
- 手動での検索・確認作業が不要になる
- TOML ファイルで管理対象を一覧管理できる
- 更新漏れを防げる
